# Public Release Matrix

## DIY/Core

Audience: users who want the governance framework and task-card workflow but
prefer to bring their own skills and rules.

Includes:

- protocol docs
- fallback task-card templates
- project integration templates
- neutral global rules
- one-click install, verification, diff, rollback, doctor/update, and task
  runner scripts
- capability registry

Excludes:

- optional skill packs, if a downstream distribution adds any
- personal skill packs
- private project registries
- machine-specific release topology

## Full Installer

Audience: users who want to recreate the high-performance multi-agent
development environment.

Includes:

- everything in DIY/Core
- required global skills
- hook normalization
- project workflow installer
- runtime installer
- update gate for public full-installer refreshes

Optional third-party skill packs are not bundled in the public Full Installer by
default. The optional pack directory is reserved for downstream or future use.

Full Installer remains public-safe: project names, user persona, sync targets,
and local paths are generated by the installer or supplied by the user.
