# 五条宪章：AGS 的设计哲学

README 把五条宪章压成了一张表。表让人记得住，却记不住为什么。这份文档补上"为什么"——每一条是被哪一类事故逼出来的，又在 AGS 里落成了哪一道闸门。

顺序不重要。它们不是从某个抽象原则推下来的，是一个 AI 编程新手在一个月里，被同一批问题反复绊倒之后，一条条捡起来的。

## 一·不迷信单一 AI、单一工具

最强的 AI 编程工具，现在大概就是 Codex、Claude Code、Gemini CLI、Cursor 这几类。它们都强，但强的地方不一样：代码风格、执行惯性、能力边界、对风险的判断偏差、成本结构，各不相同。再加上版本更新太快——这个版本顺手，下个版本可能突然变味。

所以我越来越不信"一个模型包打天下"。在我自己的流程里，方案、审查、修大 bug 交给一类模型，具体执行交给另一类，交付之后再换一个来验收——因为一个模型交付的代码，它常常查不出自己的问题。

AGS 不替代任何一个 Agent。它给这些 Agent 提供一套共同的工程秩序：谁规划，谁执行，谁审查，谁能动文件，谁只能只读，什么时候必须停下来等人确认。一台电脑只装一套，AGS 负责治理，让所有 Agent 在同一套规则下分工。

## 二·AI 听不全人话

提示词本质上还是聊天语言。中文又特别复杂，同一个词在不同语境下意思可能完全相反。现在很多 Agent 都能"优化提示词"，但再怎么优化，自然语言还是自然语言——它不是工程合同，也不是机器协议。

所以人的想法不该直接丢给执行的智能体，中间得有一道翻译。先项目预检，让智能体知道当前仓库是什么、有哪些规则、有哪些历史记忆、哪些事必须停下来问人；再进入方案形成，和人一起把目标、边界、非目标说清楚；用户确认后，才生成任务卡；任务卡再被解析成执行策略；最后过门禁，才允许真正开工。

一句话：提示词是聊天语言，任务卡才是工程合同。任务卡里必须写清目标、背景、非目标、权限模式、执行边界、验证方式和交付格式。它不是给人填的表，是把人的意志，校准成智能体能稳定执行的工程任务。

## 三·执行不是一条直线

AI 的执行能力不是一条直线。它有时天才，有时正常，有时走神，有时一本正经地胡说。所以你不能只看最后一句"我完成了"。

真正靠谱的工作流，必须把过程留下来：它接到的任务卡是什么，拿到了什么权限，门禁过没过，改了哪些文件，跑了哪些验证命令，结果是什么，最后有没有回执。

AGS 的目标不是让 AI 永不犯错——这不现实。它的目标是让错误不要悄悄发生。哪怕出错，也要让人追得到：在哪出的，为什么出，改了什么，没改什么，验证了什么，哪里还不确定。

一个会漂移的元件，你管不住它每一次输出，但你可以让它的每一次输出都被记录、被观测。能被看见的错误，才是能被修的错误。

## 四·高光判断该被沉淀

做项目越久，我越觉得真正宝贵的不是某一次模型输出，而是人在方案和架构阶段的判断：这个功能为什么不能这么做；这个边界为什么必须卡死；这个任务为什么只能只读；这个方案为什么要拆成两步；这个坑为什么下次一定要绕开。

这些东西只留在聊天记录里，很快就没了。AGS 用记忆胶囊把它们接住：项目画像、上下文记忆、任务归档、交付记录、关键判断、验证结果、未完成事项、风险说明。重开一个新对话，只要是这个项目里发生过的事，智能体就能从这里回忆起来，而不是只靠模型自带的那点上下文摘要。

AI 编程最大的损耗之一，就是上下文损耗。你明明教过它一次，换个对话，它又像失忆一样重新踩坑。记忆胶囊要做的，就是让经验从聊天里逃出来，变成项目资产。

## 五·模型搭配，干活不累

国外顶级模型确实好用，但贵。国产模型便宜大碗，完全放飞又容易不稳。AGS 的价值，是把便宜模型放进工程流程里，让它们在明确任务、明确边界、明确验收标准的前提下执行。顶级模型负责关键判断，平价模型负责大量具体工作，AGS 把整个过程管住；交付之后，再让更强的模型查漏补缺。

这不是为省钱而省钱，是 AI 编程迟早要面对的现实。很多平台都在收紧额度，不可能所有任务都全程用最贵的模型跑。

这条背后是个朴素的工程直觉：你没法让一个便宜元件变得不便宜，但你可以用一套确定的流程，让它在确定的边界里，交付确定的结果。模型能力会波动，工程流程负责兜底——这句话在这一条上最实在。

## 这套秩序背后，是一门老学问

写到这儿你大概看出来了，这五条没有一条是 AI 时代的新发明。

把一个不精确、会漂移、带噪声的元件，放进一套有反馈、有约束、有记录的系统里，让整体稳定地交付可用的结果——这是工程控制论在 1954 年就讲清楚的事。钱学森写《工程控制论》时，面对的是导弹的陀螺仪和伺服电机；今天我们面对的是会写代码、也会胡说的大模型。被控对象换了，控制的思路没变：不追求一个永不出错的元件，而是设计一个能容错的回路。

AGS 把这套老思路搬到了 AI 编程的现场。任务卡是设定值，门禁是约束，验证和回执是反馈，记忆胶囊让系统记住自己走过的路。与其说它是创新，不如说是一次迟到的常识回归。
