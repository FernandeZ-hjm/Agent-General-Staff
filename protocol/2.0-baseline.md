## 摘要

Agent Governance Suite 2.0 M0 基线冻结记录当前 `ags` CLI 命令面、协议边界、测试门禁
和 crate 结构。后续 M1-M8 以此基线为出发点，以可验证的方式增量演进。

2.0 公开版发布完整的 Rust `ags` 工具链（`crates/`、`target/`、`Cargo.toml`、`Cargo.lock`），
面向终端用户提供 task-card validation、execution policy resolution、protocol drift checking、
suite health diagnostics、bootstrap simulation、project discovery 和 scoped verification
等能力。

## 命令面（M0 冻结，M1 升级，M2 新增）

| M0 命令（冻结） | M1 命令（新增/主推） | M2 命令（新增） | 库 crate | 说明 |
|---|---|---|---|---|
| `ags task-card-validator <paths>` | `ags task validate <paths>` | — | `task-card-validator` | 任务卡格式与语义校验 |
| `ags resolve-policy <path> [--format] [--approve-writes]` | `ags policy resolve <path> [--format] [--approve-writes]` | — | `execution-policy` | 执行策略解析 |
| `ags workflow-sync-check [--source] [--targets] [--format]` | `ags sync check [--source] [--targets] [--format]` | — | `workflow-sync-check` | 协议漂移检查 |
| `ags suite-doctor [--format]` | `ags doctor [--format]` | — | `suite-doctor` | 套件健康诊断 |
| `ags bootstrap-dry-run [--format]` | `ags bootstrap --dry-run [--format]` | — | `bootstrap-dry-run` | 引导干运行模拟 |
| — | — | `ags session preflight --for <agent> [--target] [--format]` | `project-discovery` | 聚合 Agent 唤醒检查（kernel activation 入口） |
| `ags verify --scope local|full|release [--target] [--format]` | `ags-verify` | 结构化验证入口（CheckItem 模型，text/json 双格式） |

输出格式：除 `task-card-validator` / `task validate` 外，M0/M1 命令支持
`--format text|json`（clap value_parser 强校验），非法 `--format` 值由 clap
拒绝并 exit 2。

exit code 约定：`text` 和 `json` 格式 exit code 语义一致；0 = pass，1 = fail/validation failed，2 = 参数错误。

## 仓库边界

本仓库是公开可分发版本。通过 `WORKSPACE.md` 自动检测仓库身份和角色映射。
不依赖私有基础设施、私有仓库或内部服务。

边界规则：
- 公开版自包含，不依赖私有基础设施
- 默认 DIY 安装：只包含 Rust `ags` core、公开协议和模板
- 第三方技能只推荐不安装
- 用户自行选择可信来源并手动安装技能

## execution-policy resolver 规则（M1-M10）

| Rule | 说明 |
|---|---|
| M1-M3 | `ultracode` 仅 thinking intensity，不改变 permission mode、parallelism、launch args |
| M4 | Heavy 任务无 explicit write approval → downgrade 到 plan-only + confirmation gate |
| M5-M6 | read-only/plan-only 不产生 write-type launch args，strip active parallelism 和 headless |
| M7 | subagent/multi-session/agent-team 需要 Workflow authority，worktree 需要非 none authority |
| M8-M10 | 每次 downgrade 记录结构化 DowngradeReason，generic adapter 限制 permission 为 plan-only |

## 协议安全断言（workflow-sync-check 强制执行）

1. `ultracode thinking-only` — ultracode 不提升权限
2. `Heavy downgrade` — Heavy 无批准降级到 plan-only
3. `read-only/plan-only no-write` — 不产生写入类 launch args
4. `runner resolver-first` — runner 必须消费 resolved policy JSON，不从原始字段拼接参数
