# AGS 2.0 Roadmap

> **命名空间说明**: Roadmap M0-M8 是里程碑编号。
> execution-policy resolver 的 Policy M1-M10 是独立的规则编号。
> 二者在不同的命名空间中，互不相关。

Baseline: `protocol/2.0-baseline.md`（2026-06-05 冻结）

## Milestones

### M0: Baseline Freeze ✓ (current)

记录现有命令面、协议边界、测试门禁、仓库边界。冻结为后续阶段提供可验证的出发点。

**交付**：`protocol/2.0-baseline.md`

**依赖**：无
**依赖它的阶段**：M1-M8

### M1: CLI Object Model ✓ (current)

将内部 crate 风格 flat 命令重排为产品对象命令，新增/主推：
`ags task validate`、`ags policy resolve`、`ags sync check`、`ags doctor`、
`ags bootstrap --dry-run`，保留旧命令作为 hidden alias。

**交付**：
- `ags` CLI 新命令面
- help 文案以新对象命令为主
- 旧命令隐藏但仍可用
- 新旧命令 text/json 输出和 exit code 契约不破坏
- `scripts/verify.sh` 新增兼容性 smoke tests

**非目标**：不实现 M2-M8 能力，不删除旧命令

**依赖**：M0
**依赖它的阶段**：M2-M8（所有后续阶段使用新命令面）

### M2: Project Discovery / Agent Awareness ✓ (current)

`ags` 能感知当前仓库身份：项目 profile、workspace 角色、本地 memory 路径、
git remote / bare repo 映射，无需 task card 手动声明。

**交付**：
- `ags project detect` — 检测项目身份与 AGS 集成状态（suite/integrated/partial/not-integrated）
- `ags protocol status` — 检查协议文件状态、校验器入口、风险边界、review/verify/receipt 要求
- `ags agent instructions --for codex|claude-code|cursor` — 导出 Agent 专属项目说明（text/json）
- `ags session preflight --for codex|claude-code|cursor [--target <path>] [--format text|json]` — 聚合唤醒检查（kernel activation 入口，组合 project detect + protocol status + agent instructions + memory paths + stop conditions + warnings/failures + next steps；不依赖 skill governance）
- Workspace identity auto-detection（A/A1/S/B/B1 role mapping，via WORKSPACE.md 解析 + 已知路径 fallback）
- `crates/project-discovery/` 库 crate：可复用的检测/状态/指令/preflight 逻辑 + 68+ 个单元测试
- Agent 可调用的只读上下文查询（text/json 双格式，结构化 exit code）
- `ags` 通过 `cargo install --path crates/ags-cli` 安装到 `~/.cargo/bin/ags`，成为 PATH 可用的默认内核唤醒二进制

**非目标**：不启动 Agent，不执行任务；不实现 `ags run`/`ags verify`/`ags release`/`ags skill`

**依赖**：M1
**依赖它的阶段**：M3-M8

### M3: Gate / Policy Engine ✓ (private-only)

在 M2 感知基础上强化门禁：project-level protected paths、auto-downgrade 触发条件、
cross-target gate 检查。将 execution-policy resolver 规则从纯 task-card 字段扩展到
project-level 上下文。

**公开版状态**：`ags policy explain` 和 `ags policy check` 已包含在公开版中。
`ags gate check` 和 `scripts/run-task-card.sh --check-only|--dry-run` 的 gate 集成
属于私有版能力，不在公开版范围内。

**依赖**：M2
**依赖它的阶段**：M4-M8

### M4: Task Compiler ✓ (private-only)

将 task-card-compiler 从 prompt-based convention 提升为 Rust tooling 支持的确定性
编译路径：用户灵活意图 → canonical task card skeleton。

**公开版状态**：`ags task compile` 和 `crates/task-compiler/` 属于私有版能力，
不在公开版范围内。

**依赖**：M3
**依赖它的阶段**：M5-M8

### M5: Capability Registry ✓ (private-only, delivered 2026-06-05)

只读发现本仓库能力：protocol 文件、scripts、Rust crates。

**公开版状态**：`ags capability list|show` 和 `crates/capability-registry/` 属于私有版
能力，不在公开版范围内。

**依赖**：M3
**依赖它的阶段**：M6-M8

### M6: Receipt / Compliance ✓ (private-only, delivered 2026-06-05)

引入 task run receipt 作为一等 artifact，支持 generate / verify / compliance check
最小闭环。

**公开版状态**：`ags receipt generate|verify`、`ags compliance check` 和
`crates/receipt/` 属于私有版能力，不在公开版范围内。

**依赖**：M5
**依赖它的阶段**：M7-M8

### M7: Bootstrap / Doctor / Release ✓ (delivered 2026-06-05)

从 M1 的 `ags bootstrap --dry-run` 扩展到 `ags bootstrap --apply` 和
`ags doctor --repair`，加上 release manifest 校验。

**实际交付**：
- `ags bootstrap --apply --target <dir>` — 写入 public-safe payload (protocol/, scripts/) 到目标目录 (tempdir 验证通过)
- `ags bootstrap --dry-run --target <dir>` — 在任意目标上运行结构检查
- `ags bootstrap` mutual exclusion: `--dry-run` 和 `--apply` 互斥
- `ags doctor --repair --dry-run` — repair plan，显示可修复项
- `ags doctor --repair` — 执行安全修复: chmod scripts/*.sh, mkdir scripts/, cargo fmt (仅当 Cargo.toml + src/ 存在)
- `bootstrap_dry_run::plan()` / `apply()` / `verify()` — 库函数 (3 个单元测试)
- `suite_doctor::repair_plan()` / `repair()` — 库函数
- `workflow_sync_check::verify_release_manifest()` — release manifest 校验 (3 个单元测试)
- 6 个 M7 smoke tests 纳入 `scripts/verify.sh`

**边界**：
- bootstrap --apply payload 仅含 public-safe 协议文件和脚本；verify.sh 明确排除
- 写目标仅限 tempdir，未对 A/S/B/B1/A1 做真实 apply
- doctor repair 白名单: chmod scripts/*.sh, mkdir scripts/, cargo fmt on temp crate
- 不创建协议文件，不安装 hooks，不改 skill 目录
- 不做 bootstrap rollback

**非目标 (M7.1+)**：
- bootstrap --apply 对真实仓库的完整写入
- bootstrap --rollback
- doctor repair 对协议文件/hooks 的修复
- `ags release verify` CLI 命令 (library fn only)

**依赖**：M6
**依赖它的阶段**：M8

### M8.5: Verification Core (in progress)

将 `scripts/verify.sh` 的核心权威迁到 Rust：新增 `ags verify` 命令和
`crates/ags-verify/` 库 crate，提供稳定 `CheckItem` 模型（id、scope、status、
severity、evidence、remediation、command、exit_code）和 scoped 验证
（`--scope local|full|release`）。`scripts/verify.sh` 降级为兼容 wrapper。

**交付**：
- `ags verify --scope local|full|release --format text|json`
- `crates/ags-verify/` 库 crate 含 CheckItem、VerificationReport 模型
- 核心检查迁移为 Rust：cargo fmt、cargo test、cargo build、fixtures、YAML parse、session preflight、drift checks、boundary checks
- `scripts/verify.sh` 降级为 wrapper：主门禁为 `ags verify --scope full`
- 剩余 shell-only smoke tests 清单（resolver hardening、gate/policy engine、lifecycle semantics）

**非目标**：不修改 S/B/B1/A1，不改变 public-full-sanitized sanitize 边界

**依赖**：M8
**依赖它的阶段**：后续 release pipeline

### M8: Dogfood ✓ (Claude Plan Mode delivered 2026-06-05)

在 A 仓库上完成 Heavy plan-only dry-run 链路验证，并补齐真实 Claude
Plan Mode session dogfood。runner 已通过 AGS gate 调用真实 Claude Code，
并把 resolver 生成的 `--permission-mode plan` 作为 launch arg 传递给
executor。

**已交付**：
- `scripts/run-task-card.sh` executor dispatch: claude-code adapter 使用 `claude $LAUNCH_ARGS -p - < task-card` 接线 (stdin 形式，替换 M3 placeholder)
- Claude Plan Mode 探针验证: `claude --permission-mode plan -p - < task-card` (v2.1.165) 返回 READY
- Heavy plan-only dry-run 链路: task compile → validate → gate check → policy resolve → runner dry-run → receipt generate → receipt verify → compliance check（7 阶段全通过）
- 真实 Claude Plan Mode dogfood: runner 输出 `Launch args (from resolved policy, verbatim): --permission-mode plan`，Claude 返回 `AGS_M8_DOGFOOD_READY`
- 6 个 Runner Wiring smoke tests 纳入 `scripts/verify.sh`

**证据**:
- gate check → confirm, `allowed_launch_args: ['--permission-mode', 'plan']`
- runner dry-run → `--permission-mode plan` correctly flows from resolver
- real runner session → `claude --permission-mode plan -p - < task-card`
- Claude marker → `AGS_M8_DOGFOOD_READY`
- receipt verify → valid True
- compliance check → compliant True

**后续扩展 (非 M8 delivered blocker)**：
- light/medium 链路扩展
- codex-local / cursor adapter 接线
- 完整 delivery report 生成和闭环

**依赖**：M7

## 非里程碑事项（贯穿 M0-M8）

- public-full-sanitized payload 边界始终不变：公开版保留满血 AGS runtime，但不携带私人数据、本机状态、构建产物或已安装技能
- `task-card-template.md` 固定骨架不 break
- task-card-validator canonical gate 地位不变
- execution-policy resolver M1-M10 规则 backward compatible
- verify.sh 门禁始终绿灯
