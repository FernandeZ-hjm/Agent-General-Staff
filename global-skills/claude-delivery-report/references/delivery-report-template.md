# Claude Code Delivery Report Template

完成任务后，必须按本模板输出交付报告。不得只回复“完成了”“已修复”“测试通过”。

保持简洁。重量级任务的详细审计内容应放在输出物或 summary 文件里，主报告只给用户决策所需信息。

## 模板

````markdown
# 任务交付报告

## 任务状态

完成 / 部分完成 / 未完成

一句话结论：

{summary}

## 改动内容

修改文件：

- `{file_1}`：{change_summary}
- `{file_2}`：{change_summary}

新增文件 / 输出物：

- `{new_file_or_output_1}`：{purpose}
- `{new_file_or_output_2}`：{purpose}

删除文件：

- 无 / `{deleted_file}`：{reason}

## 验证结果

已运行：

```bash
{command_1}
{command_2}
```

结果：

- {verification_summary}

未验证内容：

- 无 / {unverified_item_and_reason}

## 风险提示

- {risk_1}
- {risk_2}

如果没有明显风险，写：

```text
未发现明显风险。
```

## 下一步建议

- {next_step_1}
- {next_step_2}
````

## 填写规则

- `任务状态` 只能写：完成 / 部分完成 / 未完成。
- `修改文件`、`新增文件 / 输出物`、`删除文件` 没有内容时写 `无`。
- `验证结果` 必须列出实际运行的命令；没有运行时写明原因。
- `未验证内容` 不能省略；全部验证过才写 `无`。
- `风险提示` 必须包含数据风险、回滚风险、兼容性风险、人工复核点中相关的项。
- `下一步建议` 只写最值得做的 1-3 项。
- 不要粘贴长日志；只摘取证明成功或失败的关键结果。
- 如果工作区有非本任务改动，必须在 `风险提示` 或 `下一步建议` 中说明。
