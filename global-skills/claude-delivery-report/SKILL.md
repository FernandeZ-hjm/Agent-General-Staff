# Claude Delivery Report

Use this skill when finishing an implementation, diagnosis, refactor, data task, or any Claude Code execution task generated by Codex.

The goal is not to write a long audit memo. The goal is to give the user a fast, reliable handoff:

- whether the task is complete,
- what changed,
- what was added,
- what was verified,
- what risks remain,
- what to do next.

## Workflow

1. Review the original task and acceptance criteria.
2. Review the files changed in this task.
3. Review new outputs or generated artifacts.
4. Run or report the verification commands requested by the task.
5. Check `git status --short` when inside a git repository.
6. Read `references/delivery-report-template.md`.
7. Fill the report template concisely.

## Report Rules

- Keep the main report short and scannable.
- Do not say "completed" unless verification evidence supports it.
- If verification was skipped or impossible, say so plainly.
- If no files were modified, write `无`.
- If there are unrelated working-tree changes, separate them from task-related changes.
- For heavy tasks, put detailed audit counts and long evidence in generated artifacts or summary files, then reference them from the short report.
- Never hide risks or unverified assumptions.

## Required Sections

Every report must include:

1. 任务状态
2. 改动内容
3. 新增文件 / 输出物
4. 验证结果
5. 风险提示
6. 下一步建议

Optional details can be added only when they materially help the user decide the next action.
